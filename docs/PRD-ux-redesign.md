# PRD：闪象 UX 流程重构

## 1. Summary

重构闪象从登陆到完成发布的完整用户流程。解决当前用户在生成图片后迷路、命名混乱、缺少引导的问题。核心原则：**每一步用户都清楚下一步该干什么。**

## 2. 当前问题

1. Dashboard 项目卡片点进去是详情页，无"继续制作"入口
2. Studio 生成完成后用户不知道下一步——没有发布/下载引导
3. 命名不直观：AI创作/项目库 vs 新建项目/我的项目
4. 发布页和下载混在一起，用户困惑
5. 状态显示不准确——新生成的图显示"已发布"

## 3. 目标 UX 流程

```
登陆 → Dashboard
         │
         ├─ 新建项目 → Studio（上传→选风格→生成→预览）
         │                  │
         │                  ├─ 满意 → [发布] → 发布页（选平台→发布）
         │                  ├─ 满意 → [下载] → 直接下载
         │                  └─ 不满意 → 继续调整
         │
         └─ 我的项目 → 项目详情（查看结果）
                           ├─ [继续制作] → 回 Studio
                           └─ [发布/下载] → 发布页
```

## 4. 侧边栏导航

| 英文 | 中文 | 图标 | 链接 |
|---|---|---|---|
| Dashboard | 控制台 | LayoutDashboard | /dashboard |
| New Project | 新建项目 | Plus | /studio |
| My Projects | 我的项目 | FolderOpen | /products |
| Assets | 素材中心 | Image | /assets |
| Settings | 设置 | Settings | /settings |

去掉"平台链接"（integrations），集成到设置页面的品牌预设里。

## 5. 页面状态流转

```
DRAFT → GENERATING → GENERATED → PUBLISHING → PUBLISHED
  ↓                     ↓
  └── FAILED            └── 可以继续编辑
```

- DRAFT：刚创建，还没生成
- GENERATING：正在生成中
- GENERATED：生成完成，可在 Studio 预览
- PUBLISHING：正在发布到平台
- PUBLISHED：已发布
- FAILED：生成/发布失败

去掉 REVIEW 和 APPROVED（当前没有审批流程）。

## 6. Studio 页面改动（核心）

### 当前：只有生成，没有后续动作
### 改为：生成完成后出现操作栏

生成完成后，在预览图下方显示：
- [发布] 按钮 → 跳转发布页
- [下载] 按钮 → 直接下载当前图片
- [继续调整] → 重新生成

## 7. Dashboard 项目卡片

- 显示状态标签（草稿/生成中/已完成）
- Hover 显示 [继续制作] 按钮
- 点击卡片 → 进入项目详情

## 8. 项目详情页

- 显示该项目的所有生成结果
- [继续制作] → 回 Studio
- [发布/下载] → 发布页

## 9. 发布页

保持现有逻辑：选图片 → 选平台 → 下载/发布。不需要大改。

## 10. 命名统一

| 旧名 | 新名 |
|---|---|
| AI 创作 | 新建项目 |
| 项目库 | 我的项目 |
| 平台链接 | 删除（合并到设置） |

## 11. 实施计划

### Phase 1：关键路径（先做）
1. 侧边栏重命名：AI创作→新建项目，项目库→我的项目 ✅
2. Studio 生成完成后加发布/下载操作栏
3. Dashboard 项目卡片 hover 显示继续制作 ✅
4. 项目详情页加继续制作按钮 ✅

### Phase 2：清理（后做）
5. 去掉 integrations 页（合并到设置）
6. 简化状态枚举（去掉 REVIEW/APPROVED）
7. 修复 Dashboard 状态显示逻辑
